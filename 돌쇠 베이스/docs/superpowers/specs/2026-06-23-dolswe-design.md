# 돌쇠 — 행사장 대화 AI 설계

## 목적

행사장 방문객과 음성으로 대화하는 로컬 AI. 캐릭터 "돌쇠" — 현대 한국어, 고1~2 학생 느낌, 천박함과 친근함 사이의 날것 말투(무례하지 않음). 완전 오프라인, CPU 전용 PC 1대에서 동작.

## 핵심 제약

- **CPU 전용** (GPU 없음). 무거운 비전 LLM은 느림 → 가벼운 대안 우선, 모든 무거운 작업은 비동기로 메인 대화를 막지 않음.
- **완전 오프라인.** 외부 API/인터넷 사용 안 함. 로컬 Ollama + 로컬 모델만.
- **반응성 우선.** "허술해도 안 끊기면 자연스럽다" — 정확도보다 대화 연속성.
- **접근성.** 실시간 자막 + 타이핑 입력으로 청각장애인/시끄러운 환경 대응.

## 기능

1. **음성 대화** — STT로 듣고, LLM으로 페르소나 응답 생성, TTS로 말함.
2. **카메라 상황 인식** — 사람 감지(다가오면 선제 발화), 손흔들기 제스처, 옷/장면 묘사를 대화에 반영.
3. **접근성** — 실시간 자막(현재 문장, 약 3초 표시), 키보드 타이핑 입력.
4. **말하는 시각 효과** — Profile.svg에 물결(ripple) 이펙트. 발화 중 동심원 + 스케일 펄스, TTS 음량 진폭에 반응. idle엔 잔잔한 숨쉬기.

## 아키텍처

멀티스레드 파이프라인 + 공유 컨텍스트. 각 작업은 독립 워커로 병렬 → 병목 분리.

```
[웹캠/마이크 캡처]
       │
   ┌───┴────────────┬──────────────┐
   ▼                ▼              ▼
[비전-빠름]      [STT]         [비전-느림(async)]
MediaPipe      faster-whisper   moondream
   │                │              │
   └──── 공유 컨텍스트 dict ◄───────┘
            (사람수, 제스처, 색, 장면묘사)
                    │
            [두뇌: Ollama LLM] ◄── 타이핑 입력
              토큰 스트리밍
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
   [TTS: Piper]            [UI 자막/물결]
   문장 단위 큐             pywebview
```

### 컴포넌트

| 컴포넌트 | 모델/도구 | 역할 | 인터페이스 |
|---|---|---|---|
| 캡처 | OpenCV, sounddevice | 프레임·오디오 수집 | 프레임/오디오 버퍼 제공 |
| 비전-빠름 | MediaPipe | 얼굴/사람 감지, 손흔들기 제스처, 옷 주요색(OpenCV) | 공유 컨텍스트 갱신, 이벤트 발행 |
| 비전-느림 | moondream (Ollama) | 새 사람 트리거 시 장면 1회 묘사 (async) | 공유 컨텍스트 갱신 (지연 도착 허용) |
| STT | faster-whisper base int8 (ko) | VAD 발화 구간 → 텍스트 + 실시간 자막 | 텍스트 큐, 자막 이벤트 |
| 두뇌 | Ollama: EXAONE-3.5-2.4B (한국어) | 페르소나 프롬프트 조립 + 응답 토큰 스트리밍 | 입력 큐 소비, 문장 스트림 출력 |
| TTS | Piper (ko) | 문장 단위 음성 합성·재생, 음량 진폭 노출 | 문장 큐 소비, 진폭 이벤트 |
| UI | pywebview (HTML/SVG/CSS/JS) | 단일 네이티브 창. 물결 SVG, 자막, 타이핑 입력 | Python↔JS 브리지 |

비전-느림(moondream)은 우선 시도. 너무 느리면 OpenCV 주요색 추출로 다운그레이드.

## 데이터 흐름

1. 입력: STT 텍스트 **또는** 타이핑 입력 → 두뇌 입력 큐.
2. 두뇌가 응답 시점에 공유 컨텍스트(사람수/제스처/색/묘사)를 읽어 프롬프트 조립.
3. 새 사람 감지 이벤트 → 두뇌 선제 발화 트리거.
4. LLM 토큰 스트리밍 → 문장 완성될 때마다 TTS 큐 + UI 자막 동시 전송.
5. TTS 순차 재생, 음량 진폭을 UI 물결로 전달.

## 지연 전략 (안 끊기게)

- VAD로 발화 끝 감지 → 즉시 STT.
- LLM 토큰 스트리밍 → 첫 문장 완성 즉시 TTS 시작(전체 응답 대기 안 함).
- TTS 문장 큐 순차 재생.
- moondream 등 무거운 비전은 항상 async, 메인 대화 절대 차단 안 함.
- **바지인(barge-in):** 돌쇠 발화 중 사용자가 말하면 → TTS 중단, 새 입력 처리.

## 에러/엣지 케이스

- 카메라 없음 → 비전 기능 끄고 음성+타이핑만.
- 마이크 없음 → 타이핑 전용 모드.
- 모델 로드 실패 → UI에 표시, 가능한 기능만 동작.
- 큐 크기 제한 → 오래된 프레임/입력 폐기 (지연 누적 방지).
- 발화자 다수 → 가장 가깝거나 큰 얼굴 1명 기준.

## 페르소나

- 현대 한국어, 고1~2 학생 톤. 날것·투박하지만 무례하지 않음.
- 짧고 직설적. 카메라 컨텍스트 자연스럽게 반영("빨간 옷이네", "둘이 왔네").
- 시스템 프롬프트로 톤 고정, 욕설/공격성 금지 가드레일 포함.

## 기술 스택

- Python.
- STT: faster-whisper (base, int8, ko).
- LLM: Ollama + EXAONE-3.5-2.4B (대안: qwen2.5:3b).
- 비전: MediaPipe(감지/제스처), OpenCV(색), moondream(묘사, async/폴백 가능).
- TTS: Piper (ko 음성).
- UI: pywebview (단일 네이티브 창, HTML/SVG/CSS/JS, Python↔JS 브리지).
- 전부 오프라인.

## 비목표 (YAGNI)

- 풀 아바타/3D/립싱크 모델 — 안 함. Profile.svg 물결만.
- 클라우드/API — 안 함.
- 다국어 — 한국어만 (요구 시 추후).
- 다중 동시 대화 세션 — 1명 기준.
